Part 16: Polymorphism with toString() method

1. What are the advantages of Polymorphism?
- Extensibility: It makes expanding the system very easy. If we create a new class inheriting from Media later (e.g., VinylRecord), we only need to override the toString() method for it. The loop printing the list above won't require a single line of code change.

- Simplicity and Clean Code: It allows us to handle a collection of different objects (CD, DVD, Book) through a single common interface (the parent type Media). This helps avoid writing multiple nested if-else or switch-case statements to check the object type.

2. How is Inheritance useful to achieve Polymorphism in Java?
- Inheritance establishes an "IS-A" relationship (e.g., a Book is a Media). Thanks to this relationship, Java allows Upcasting - which is using a parent class reference variable (like Media m) to point to a child class object (like new Book()).

- Inheritance also provides the mechanism for child classes to Override parent class methods. Without Inheritance, we cannot apply Polymorphism.

3. What are the differences between Polymorphism and Inheritance in Java?
- Inheritance is Structural. It is about building a hierarchy between classes and sharing attributes and methods from parent to child for the purpose of code reusability.

- Polymorphism is Behavioral. It allows a single message (a method call) to trigger different behaviors depending on the actual object receiving that message (for example, calling the exact same toString() method, but a Book prints the author's name, while a CD prints the artist's name).
